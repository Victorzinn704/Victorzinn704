## O que essas ferramentas já sustentam no Desk Imperial

- **Oracle Cloud** e **Linux** como alvo de runtime independente da aplicação gerenciada
- **Grafana**, **Prometheus** e trilha **OSS** para observabilidade da operação
- **SonarQube** para reduzir regressão, ruído e bugs silenciosos
- **Redis**, **Neon** e **Prisma** como base de dados quente + persistência relacional
- **Docker** como camada de padronização local e futura estrutura de runtime
- **PWA** para operação mobile do dono e da equipe em contexto real

---

## Foco atual

- Evolução do **Desk Imperial** como produto vivo
- Qualidade de código com **CI, testes e SonarQube**
- Observabilidade com **OpenTelemetry + Alloy + Grafana + Prometheus**
- Runtime independente com **Oracle Cloud + Neon**
- Performance de fluxo operacional, tempo real e UX de dashboard/mobile
